# shift-period-config Delta Spec

## 変更概要

シフト期間設定UIから「クリア」ボタンを削除する。保存ボタンのみを残し、誤操作リスクを低減する。

## 削除する要件（暗黙的に存在していたもの）

以下の要件は `spec.md` には明示されていなかったが、テストコードおよびUIとして実装されていた：

- 「シフト作成期間をクリア（削除）できる」— UIから誤操作リスクがあるため削除する
  - 根拠: proposal.md の「誤操作リスク」「不要な機能」セクションを参照
  - 削除されるテストケース: `'クリアボタンを押すとシフト期間が削除される'`（`SettingsPage.test.tsx` 115〜128行目）

## 変更しない要件

以下の要件は本変更の影響を受けない（既存 `spec.md` のままとする）：

- シフト作成期間（開始日・終了日）を設定できる
- 開始日は終了日以前でなければならない（バリデーション）
- 期間は5週間（35日）以内でなければならない（バリデーション）
- 期間が未設定の場合は当月16日〜翌月15日をデフォルトとして扱う

## UIの変更

### 変更前

シフト期間設定フォームのボタン行に「保存」と「クリア」の2つのボタンが表示される。

- **保存**: 入力した期間をLocalStorageに保存する
- **クリア**: LocalStorageの期間データを削除し、入力フォームを空にする

### 変更後

シフト期間設定フォームのボタン行に「保存」ボタンのみが表示される。

- **保存**: 入力した期間をLocalStorageに保存する（変更なし）

## 削除するUI要素

- 「クリア」ボタン（テキストコンテンツ「クリア」を持つ `<button>` 要素、`SettingsPage.tsx` ボタン行）
- `handleClearPeriod` 関数（`setPeriodStart('')` / `setPeriodEnd('')` のフォームリセット動作を含む内部実装）
- `clearShiftPeriod` の分割代入を `useShiftPeriod` から削除する（UIから参照しなくなるためデッドコードとなる）

なお、`useShiftPeriod` フック内の `clearShiftPeriod` 関数本体は本変更では削除しない（フック側への影響を最小化する）。

## テストへの影響

- `SettingsPage.test.tsx` の「クリアボタンを押すとシフト期間が削除される」テストケースを削除する
  - クリアボタン自体が存在しなくなるため、このテストは不要となる
