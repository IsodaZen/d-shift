## Why

現在のシステムでは、必要人数が不足した場合に「ヘルプスタッフ要」のアラートを表示するが、ヘルプスタッフの実体（誰がいつ稼働可能か）を管理する機能がない。以下の課題がある。

1. **ヘルプスタッフの手配状況が不透明**: アラートで不足人数は分かるが、誰にヘルプを依頼できるかをシステム上で管理できない。管理者は別途手帳やメモで管理している。
2. **自動生成にヘルプスタッフが参加しない**: 自動シフト生成では通常スタッフのみが割り当て対象であり、ヘルプスタッフを含めた最適化ができない。不足分は手動でアサインする必要がある。
3. **不足判定が不正確**: ヘルプスタッフの稼働可能状況を考慮せずに「ヘルプ要」を表示しているため、実際にはヘルプスタッフで充足可能な場合でもアラートが出る。

ヘルプスタッフを独立したエンティティとして管理し、稼働可能日付をカレンダーUIで個別設定できるようにする。自動生成アルゴリズムにヘルプスタッフを組み込み、通常スタッフとともに全体最適化しつつ、ヘルプスタッフのアサイン数を最小化する。

## What Changes

### ヘルプスタッフエンティティの新設

通常の `Staff` とは別に `HelpStaff` エンティティを新設する。

```ts
interface HelpStaff {
  id: string
  name: string
  availableSlots: TimeSlot[]    // 出勤可能時間帯（午前/午後/夕方）
  availableDates: string[]      // 稼働可能な日付（YYYY-MM-DD形式）
  usesParking: boolean          // 駐車場利用フラグ
}
```

通常スタッフとの違い:
- **`availableDates`**: 稼働可能な日付を個別指定する（通常スタッフの「希望休」の逆の概念）
- **週上限出勤回数なし**: ヘルプスタッフは稼働可能日に出勤するため、週上限の概念を持たない
- **希望休なし**: `availableDates` で稼働可能日を指定するため、希望休は不要

### 設定ページに「ヘルプスタッフ」タブを追加

SettingsPage に新しいタブ「ヘルプスタッフ」（URL: `/settings/help-staff`）を追加する。

**タブ内の機能:**
- ヘルプスタッフの一覧表示（名前・出勤可能時間帯・駐車場利用）
- ヘルプスタッフの追加・編集・削除
- カレンダーUIによる稼働可能日付の設定（シフト期間内の日付を選択）

### 自動生成アルゴリズムの拡張

既存の自動生成アルゴリズムを拡張し、ヘルプスタッフを割り当て対象に含める。

**全体最適化方針:**
- 通常スタッフとヘルプスタッフを統合して割り当て候補とする
- ただしヘルプスタッフにはペナルティ（優先度低下）を与え、通常スタッフで充足可能な場合はヘルプスタッフをアサインしない
- 具体的には、ソート時に「通常スタッフを先、ヘルプスタッフを後」とし、ヘルプスタッフは通常スタッフだけでは充足できない場合のみアサインされる

**ヘルプスタッフの強制制約:**
- 稼働可能日付（`availableDates`）に含まれない日にはアサインしない
- 出勤可能時間帯（`availableSlots`）のみにアサインする
- 駐車場が必要な場合は空き枠がなければアサインしない
- 出勤日には利用可能な全時間帯にアサインする（通常スタッフと同じ制約）

### ヘルプスタッフアラートの改善

ヘルプスタッフの稼働可能状況を考慮したうえで、なおヘルプが必要な場合のみアラートを表示する。

- **変更前**: 通常スタッフの出勤可能人数のみで不足を判定
- **変更後**: 通常スタッフ＋稼働可能なヘルプスタッフの合計で不足を判定

## Capabilities

### New Capabilities

- `help-staff-management`: ヘルプスタッフの登録・編集・削除・一覧表示。稼働可能日付のカレンダーUI設定。設定ページの「ヘルプスタッフ」タブ

### Modified Capabilities

- `auto-shift-generation`: ヘルプスタッフを割り当て候補に含め、通常スタッフ優先の全体最適化アルゴリズムに拡張する
- `help-staff-alert`: ヘルプスタッフの稼働可能状況を考慮した不足判定に変更する
