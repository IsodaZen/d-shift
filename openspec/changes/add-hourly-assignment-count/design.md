## Context

現在の `ShiftTable` コンポーネント（`src/components/ShiftTable.tsx`）は、列ヘッダーに「不足N」インジケーターを表示するが、各時間帯のアサイン数・必要人数の具体値は表示していない。シフト担当者が充足状況を確認するには全セルを目視でカウントする必要がある。

本変更では `ShiftTable` コンポーネントのみを修正し、時間帯ごとのアサイン数集計行を追加する。既存の不足インジケーターとの重複や混乱を避けつつ、必要最小限の変更でこの情報を追加する。

## Goals / Non-Goals

**Goals:**
- シフト表の時間帯ごとに「アサイン数 / 必要人数」を表示する集計行を追加する
- 不足時はオレンジ系、充足時はグレー系で色分け表示する
- 既存の不足インジケーターや他の表示要素に影響を与えない

**Non-Goals:**
- 集計行のクリックインタラクションや編集機能の追加
- ヘルプスタッフのアサインを集計行に含めること（通常スタッフのみ）
- 不足インジケーターの削除・変更

## Decisions

### 決定1: 集計行をtbodyの先頭（スタッフ行の直前）に配置する

**選択肢:**
- A. `<thead>` 内の追加行として配置
- B. `<tbody>` の先頭行（スタッフ行の直前）として配置  ← 採用
- C. `<tfoot>` として最下部に配置

**理由:** テーブルの `<thead>` はスクロール固定のためにスタイリングがされており、行を追加するとレイアウトが崩れるリスクがある。最下部（tfoot）では多人数のスタッフ一覧時にスクロールしないと確認できない。`<tbody>` 先頭（スタッフ行の上）に置くことで、ヘッダー直下に集計情報を自然に配置できる。

### 決定2: 時間帯ごとに独立した1行ずつ表示する

**選択肢:**
- A. 時間帯ごとに1行（午前行・午後行・夕方行の3行）← 採用
- B. 各列ヘッダーセル内に縦に3行まとめて表示

**理由:** ヘッダーセル内に追加すると列ヘッダーの高さが増してレイアウトが複雑になり、既存のインジケーターとの干渉が発生しやすい。独立した行として表示することで、可読性が高くテスト容易性も優れる。

### 決定3: 左端セルに時間帯ラベルを表示する

集計行の最左列（sticky固定列）には時間帯のラベル（「午前」「午後」「夕方」）を表示し、各日付の列には「アサイン数/必要人数」を表示する。

### 決定4: 集計対象は通常スタッフのアサインのみとする

ヘルプスタッフのアサインは集計に含めない。理由は、ヘルプスタッフはシフト枠の必要人数を充足するためではなく補助的に使われる性質があり、混在すると充足状況の判断が複雑になるため。

## Risks / Trade-offs

- [リスク] 集計行を追加することで、大人数スタッフ環境では縦スクロールが増える → tbody先頭配置のため許容範囲内
- [リスク] getRequiredCount が undefined の場合（prop未渡し時）に必要人数が表示できない → 既存コードと同様に0扱いとし、「N/-」形式で表示するフォールバックを実装する

## Migration Plan

既存のコンポーネントインターフェース（props）は変更しない。`ShiftTable.tsx` 内部のみの変更であり、呼び出し元への影響はない。
