## ADDED Requirements

### Requirement: 評価基準2 — 超過人数の合計最小化

システムは、全期間の全(日, 時間帯)ペアにおける超過人数の合計（`Σ max(0, アサイン人数 - 必要人数)`、必要人数 > 0 のペアのみ対象）を第2評価基準として使用しなければならない（SHALL）。この基準は評価基準1（不足ピーク）が同値の場合にのみ比較に使用される。

**変更後の評価基準全体順序（辞書式、小さいほど良い）:**

1. 不足人数のピーク最小化（評価基準1）— 変更なし
2. 超過人数の合計最小化（評価基準2）← 本変更で追加
3. スタッフ負担の公平性（評価基準3、旧評価基準2）
4. 駐車場ピーク利用数の最小化（評価基準4、旧評価基準3）

（注: 実装上は不足合計・ヘルプスタッフ日数も内部基準として存在するが、仕様の評価基準番号は上記4基準のみを対象とする）

#### Scenario: 超過のない解が優先される
- **GIVEN** 解Aはある(日, 時間帯)ペアで1人超過している、解Bは超過なし
- **AND** 両解の評価基準1（不足ピーク）が同値
- **WHEN** 両解を評価基準2で比較する
- **THEN** 解Bが優先される（超過合計 0 < 1）

#### Scenario: 超過合計が小さい解が優先される
- **GIVEN** 解Aの超過合計が3（複数スロットの超過を合算）、解Bの超過合計が1
- **AND** 両解の評価基準1が同値
- **WHEN** 両解を評価基準2で比較する
- **THEN** 解Bが優先される（超過合計が小さい）

#### Scenario: 評価基準1が異なる場合、評価基準2は比較に使用されない
- **GIVEN** 解Aの最大不足が1人・超過合計が0、解Bの最大不足が0人・超過合計が5
- **WHEN** 両解を辞書式比較する
- **THEN** 解Bが優先される（評価基準1が優先）

#### Scenario: 不足解消のために超過が増加する解が選択される
- **GIVEN** 解Aは最大不足が0人・超過合計が2、解Bは最大不足が1人・超過合計が0
- **WHEN** 両解を辞書式比較する
- **THEN** 解Aが優先される（評価基準1が優先されるため、超過合計が多くても不足が少ない解を採用）

#### Scenario: 必要人数が0のスロットは超過計算から除外される
- **GIVEN** ある日の夕方スロットの必要人数が0で、そのスロットにスタッフが1人アサインされている
- **AND** その他のすべてのスロットはアサイン数が必要人数以下
- **WHEN** excessTotal を計算する
- **THEN** 夕方スロットは計算対象から除外され、excessTotal = 0 である

#### Scenario: 必要人数を超えたスロットの超過人数が正しく合算される
- **GIVEN** スロット1の必要人数が2でアサイン数が3（超過1）、スロット2の必要人数が1でアサイン数が3（超過2）
- **AND** その他のスロットはアサイン数が必要人数以下
- **WHEN** excessTotal を計算する
- **THEN** excessTotal = (3-2) + (3-1) = 3 である

## MODIFIED Requirements

### Requirement: 評価基準2 — スタッフ負担の公平性

システムは、各通常スタッフの残余容量（期間内の全週にわたる週上限出勤回数の合計 - 実出勤日数）の母分散（N で割る）を第3評価基準として使用しなければならない（SHALL）。この基準は評価基準1と2がともに同値の場合にのみ比較に使用される。ヘルプスタッフはこの基準の計算対象から除外する（ヘルプスタッフには週上限が存在しないため）。

#### Scenario: 分散が小さい解が優先される
- **GIVEN** 解Aのスタッフ残余容量が [2, 0, 3]（母分散 = 1.56）
- **AND** 解Bのスタッフ残余容量が [1, 1, 2]（母分散 = 0.22）
- **AND** 両解の評価基準1と2が同値
- **WHEN** 両解を評価基準3で比較する
- **THEN** 解Bが優先される（母分散が小さい）

#### Scenario: 評価基準1または2が異なる場合、評価基準3は比較に使用されない
- **GIVEN** 解Aの最大不足が0人・超過合計が0・母分散が0.1
- **AND** 解Bの最大不足が0人・超過合計が1・母分散が0.0
- **WHEN** 両解を辞書式比較する
- **THEN** 解Aが優先される（評価基準2が優先）

### Requirement: 評価基準3 — 駐車場ピーク利用数の最小化

システムは、全期間の各日における駐車場利用スタッフ数の最大値を第4評価基準として使用しなければならない（SHALL）。駐車場利用スタッフ数は、その日に出勤する `usesParking: true` のスタッフの人数である。この基準は評価基準1と2と3がすべて同値の場合にのみ比較に使用される。

#### Scenario: 駐車場ピークが低い解が優先される
- **GIVEN** 解Aの最大日次駐車場利用数が5台、解Bの最大日次駐車場利用数が4台
- **AND** 両解の評価基準1・2・3が同値
- **WHEN** 両解を評価基準4で比較する
- **THEN** 解Bが優先される（ピーク利用数が小さい）

#### Scenario: 駐車場利用を分散する解が優先される
- **GIVEN** 解Aは1日に5台・別の日に3台、解Bは両日とも4台
- **AND** 両解の評価基準1・2・3が同値
- **WHEN** 両解を評価基準4で比較する
- **THEN** 解Bが優先される（ピーク = 4 < 5）
