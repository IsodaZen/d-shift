# コーディング規約

## **MUST**: TDD（テスト駆動開発）原則

> **本プロジェクトへのすべての実装は、TDD（Test-Driven Development）の原則に従わなければならない。**

### Red-Green-Refactorサイクル

1. **Red（失敗するテストを書く）**: 実装前に、期待する振る舞いを表すテストを先に書く
2. **Green（テストを通す）**: テストが通る最小限のコードを実装する
3. **Refactor（リファクタリング）**: テストが通ることを確認しながらコードを整理する

### TDD実践ルール

- **テストなしのコード追加禁止**: 新しいロジック・関数・コンポーネントを追加する際は、必ず対応するテストを先に作成すること
- **テストファイルの配置**: テスト対象ファイルと同じディレクトリに `*.test.ts` / `*.test.tsx` として配置
- **テストの粒度**: ユニットテストを基本とし、1つのテストケースは1つの振る舞いを検証する
- **モックの利用**: 外部依存（localStorage等）はモック化してテストの独立性を保つ

## コーディングスタイル

- コンポーネントは関数コンポーネント（`const Foo: React.FC = () => {}`）
- ファイル名はコンポーネントはPascalCase（`ShiftTable.tsx`）、それ以外はcamelCase
- 型定義は`src/types/`に集約
- コメントは日本語で記述
